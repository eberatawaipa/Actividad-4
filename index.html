1.¿Qué es un Dato?

Un dato es una representación simbólica de un hecho, un concepto o una instrucción que se puede almacenar, procesar y transmitir. Los datos son elementos brutos que, por sí mismos, no tienen significado.

2.¿Qué es información?

La información es el conjunto de datos procesados y organizados de tal manera que tienen un significado y son útiles para la toma de decisiones. La información es el resultado del procesamiento de los datos.

3.¿Diferencia entre dato e información?

Dato: Elemento básico, sin interpretación ni contexto.

Información: Conjunto de datos procesados y organizados con significado y contexto.

4.¿Tipos de datos?

Datos cualitativos: Describen cualidades o características.

Datos cuantitativos: Representan cantidades y pueden ser medidos

Datos estructurados: Datos organizados en un formato fijo (ej. tablas de bases de datos).

Datos no estructurados: Datos sin un formato predefinido (ej. correos electrónicos, videos).

5.¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear un modelo visual de una base de datos que define la estructura lógica de los datos y sus relaciones. Este modelo ayuda a entender cómo los datos se organizan y se gestionan.

6.¿Tipos de bases de Datos, según:

Variabilidad:

    Estáticas: Datos que no cambian con el tiempo.

    Dinámicas: Datos que se actualizan y modifican constantemente.

Contenido:

    Bases de datos bibliográficas: Contienen referencias y resúmenes de artículos, libros, etc.

    Bases de datos de texto completo: Almacenan textos completos de documentos.

    Bases de datos numéricas: Contienen datos numéricos, estadísticas.

    Bases de datos multimedia: Almacenan imágenes, audio, video.

7.¿Qué es el modelo relacional?

El modelo relacional es un enfoque para gestionar bases de datos que organiza los datos en tablas (relaciones) que se pueden vincular mediante claves primarias y foráneas. Fue propuesto por Edgar F. Codd.

8.¿Describa los elementos del Diagrama entidad Relacional?

  • Entidades: Representan objetos o conceptos del mundo real con existencia propia.

  • Atributos: Propiedades o características de las entidades.

  • Relaciones: Asociaciones entre entidades.

  • Llaves primarias: Identificadores únicos de entidades.

  • Llaves foráneas: Campos que establecen relaciones entre entidades.

    • 9.¿Qué es una entidad?

    Una entidad es un objeto o concepto del mundo real que tiene una existencia independiente en el contexto del sistema de información. Por ejemplo, una persona, un producto, un evento.

    10.¿Qué es un atributo y sus tipos?

    Un atributo es una propiedad o característica de una entidad.

    Tipos:

  • Simples: No se pueden dividir (ej. edad).

  • Compuestos: Se pueden dividir en subpartes (ej. dirección: calle, ciudad, código postal).

  • Monovalorados: Contienen un solo valor (ej. número de seguro social).

  • Multivalorados: Contienen múltiples valores (ej. números de teléfono).

    • 11.¿Qué es una relación?

    Una relación es una asociación entre dos o más entidades. Representa cómo interactúan o están vinculadas las entidades en el modelo de datos.

    12.¿Qué son las llaves primarias, foránea y sus características?

  • Llave primaria: Un atributo o conjunto de atributos que identifica de manera única a una entidad. Debe ser único, no nulo y estable.

  • Llave foránea: Un atributo o conjunto de atributos que crea un vínculo entre dos tablas. Es una referencia a la llave primaria de otra tabla, permitiendo así la integridad referencial.

    • 13.¿Qué son las reglas del Negocio?

    Las reglas del negocio son condiciones y restricciones que definen o limitan ciertos aspectos de la estructura de datos, las operaciones y las estrategias dentro de una organización. Aseguran que los procesos de negocio se ejecuten correctamente.

    14.¿Qué es la cardinalidad o mapeo?

    La cardinalidad describe el número de instancias de una entidad que se pueden asociar con una instancia de otra entidad en una relación. Los tipos comunes de cardinalidad son:

  • Uno a uno (1:1): Una entidad A se asocia con una sola entidad B.

  • Uno a muchos (1 ): Una entidad A se asocia con muchas entidades B.

  • Muchos a muchos (M ): Muchas entidades A se asocian con muchas entidades B.
